Install, configure, and run your first Kubernetes cluster export

Prerequisites

Before you can use Kalco, ensure you have the following:

Kubernetes Access

Valid kubeconfig file or in-cluster access to a Kubernetes cluster

Git (Optional)

For version control and change tracking functionality

Terminal Access

Command line interface to run Kalco commands

Installation

Choose the installation method that best fits your environment:

Quick Install (Recommended)

Linux/macOS 
curl -fsSL https://raw.githubusercontent.com/graz-dev/kalco/master/scripts/install.sh | bash

This script automatically downloads and installs the latest version of Kalco.

Homebrew

macOS/Linux 
brew install graz-dev/tap/kalco

Install via Homebrew for easy updates and management.

Build from Source

Source 
git clone https://github.com/graz-dev/kalco.git
cd kalco
go mod tidy
go build -o kalco ./cmd

Build from source for the latest development version.

First Run

Once Kalco is installed, you can start using it immediately:

1

Verify Installation

kalco --version

Check that Kalco is properly installed and accessible.

2

Check Help

kalco --help

Explore available commands and options.

3

Run First Export

kalco export --output-dir ./my-cluster

Export your first cluster snapshot to a local directory.

Configuration

Kalco works out of the box, but you can customize its behavior:

Output Directory

kalco export --output-dir /path/to/output

Specify where to save exported resources.

Kubeconfig

kalco export --kubeconfig ~/.kube/config

Use a specific kubeconfig file.

Git Integration

kalco export --git-push --commit-message "Weekly backup"

Automatically commit changes and push to remote.

Common Examples

Here are some practical examples of how to use Kalco:

Basic Export

# Export all resources
kalco export

# Export to specific directory
kalco export -o ./cluster-backup

Git Integration

# Initialize git repo and commit
kalco export --git-push

# Custom commit message
kalco export --commit-message "Production backup $(date)"

Validation

# Validate exported resources
kalco validate

# Generate validation report
kalco validate --output html

Troubleshooting

Common issues and their solutions:

Permission Denied

Error: permission denied

Solution: Ensure you have access to the Kubernetes cluster and the output directory is writable.

Kubeconfig Not Found

Error: no configuration has been provided

Solution: Set KUBECONFIG environment variable or use --kubeconfig flag.

Git Repository Issues

Error: not a git repository

Solution: Initialize git in the output directory or use --no-git flag.
